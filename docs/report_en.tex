\begin{abstract}
This research investigates three Low-Rank Adaptation (LoRA) variants---Standard LoRA, FFA-LoRA, and RoLoRA---within a Federated Learning (FL) framework optimized for low-end hardware (NVIDIA RTX 3050). Using the Qwen2.5-1.5B model on the AG News classification task, we analyze the trade-offs between communication efficiency and model performance under varying data heterogeneity. Our results demonstrate that while Standard LoRA provides a robust baseline (87.53\% accuracy), RoLoRA and FFA-LoRA achieve communication savings of up to 62.8\% while eliminating non-linear aggregation bias.
\end{abstract}

\section{Introduction}
As Large Language Models (LLMs) scale, traditional fine-tuning becomes impractical for decentralized edge devices. Federated Learning (FL) combined with Parameter-Efficient Fine-Tuning (PEFT) like LoRA offers a solution. However, aggregating two low-rank matrices ($A$ and $B$) simultaneously in Standard LoRA introduces non-linear errors during FedAvg. This study evaluates alternative LoRA strategies to mitigate these biases and reduce bandwidth requirements.

\section{Methodology}
\subsection{Experimental Setup}
We use a single RTX 3050 (6GB VRAM) to simulate 5 clients. Models are loaded using 4-bit NormalFloat (NF4) quantization. Gradient checkpointing and sequential processing are employed to maintain memory stability.

\subsection{LoRA Variants}
\begin{itemize}
    \item \textbf{Standard LoRA}: Updates both matrices $A$ and $B$.
    \item \textbf{FFA-LoRA}: Matrix $A$ is frozen with random initialization; only $B$ is updated.
    \item \textbf{RoLoRA}: Matrices $A$ and $B$ are updated alternately across global rounds.
\end{itemize}

\subsection{Non-IID Modeling}
Data is partitioned using Dirichlet distribution with $\alpha \in \{10.0, 0.5, 0.1\}$, representing IID to extreme Non-IID scenarios.

\subsection{Aggregation Bias Analysis}
Standard LoRA exhibits a relative bias of 0.0089 due to the quadratic nature of the $\text{avg}(B_k A_k)$ term. In contrast, FFA-LoRA and RoLoRA maintain a bias of exactly \textbf{0} by linearizing the aggregation through single-sided updates.

\section{Discussion}
The results highlight a clear Pareto front where RoLoRA provides an optimal balance. We observed that extreme Non-IID ($\alpha=0.1$) significantly degrades performance for all PEFT methods, suggesting that more advanced local training techniques (e.g., proximal terms) might be necessary for heterogeneous adapter aggregation.

\section{Conclusion}
This study confirms that LLM-based FL is viable on consumer GPUs. RoLoRA is recommended for scenarios requiring a balance of speed and capacity, while FFA-LoRA is ideal for ultra-low bandwidth environments.
